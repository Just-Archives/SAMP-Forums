I did a quick run through the bug list and didn't noticed this one. ( It has been reported before, I cannot find the thread though )

* If you are in a vehicle, and if you have a /v < vehicle id > command, once you use the command, your radio stations will stop working until you get out of that vehicle.

I'm in a vehicle, i type: /v < vehicle id >, the bug starts. Here's a quick command:
pawn Код:
CMD:v(playerid, params[])
{
    if(IsPlayerInAnyVehicle(playerid)) return SendClientMessage(playerid, COLOR_RED, "* You must leave this vehicle to spawn a new one!"); // This could be added to prevent the bug ;s

    new
        sCar,
        Vehicle_ID,
        Float:v_Pos[4],
        Int;

    if(isnull(params)) return SendClientMessage(playerid, COLOR_RED, ""#CRED"USAGE: "#CORANGE"/Car < CarID | Model >");
    sCar = GetVehicleModelIDFromName(params);
    if(sCar < 400 || sCar > 611) return SendClientMessage(playerid, COLOR_RED, "* Invalid vehicle model!");

    GetPlayerPos(playerid, v_Pos[0], v_Pos[1], v_Pos[2]);
    GetPlayerFacingAngle(playerid, v_Pos[3]);
    Int = GetPlayerInterior(playerid);
    Vehicle_ID = CreateVehicle(sCar, v_Pos[0], v_Pos[1], v_Pos[2], v_Pos[3], -1, -1, -1);

    SetVehicleVirtualWorld(Vehicle_ID, GetPlayerVirtualWorld(playerid));

    PutPlayerInVehicle(playerid, Vehicle_ID, 0);
    LinkVehicleToInterior(Vehicle_ID, Int);
    return 1;
}
It's either a bug or bad code usage.
